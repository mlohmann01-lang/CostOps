Outcome Ledger Integration Sprint

## Objective

The canonical Outcome Ledger schema now exists.

Do **not** redesign the Outcome Ledger UI yet.

This sprint is about replacing the existing synthetic/demo calculations with the new canonical model while preserving the current page structure.

The goal is:

```text
Canonical Outcome Records
        ↓
Ledger Events
        ↓
Summary Helpers
        ↓
Existing Outcome Ledger Page
```

Once complete, the next sprint can redesign the page with confidence.

---

# Core Principle

The Outcome Ledger page must no longer derive values independently.

It must become a pure presentation layer over:

```text
OutcomeRecord
OutcomeLedgerEvent
buildOutcomeLedgerSummary()
calculateOutcomeLeakage()
buildOutcomeProofTimeline()
```

No duplicate calculations.

No hardcoded lifecycle maths.

No parallel demo structures.

---
