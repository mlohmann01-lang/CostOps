# Certen Website / Onboarding Fix — P1 to P5

## Goal

Fix the public Certen flow so the site no longer dead-ends at Microsoft 365 connection, supports account creation, allows Demo/Live switching, and clearly positions the M365 review as the wedge into broader Technology Economic Control.

---

# P1 — Fix onboarding flow

## Current problem

`Run Free Exposure Review` and `Get started` currently send users toward a Microsoft 365 connect screen that fails with:

> Microsoft 365 connection is not configured for this environment.

This makes the product feel broken.

## Required new flow

```text
Landing Page
  ↓
Run Free Exposure Review
  ↓
Create Account / Request Access
  ↓
Choose Workspace Mode: Demo or Live
  ↓
Connect Microsoft 365
  ↓
Generate Exposure Report
  ↓
Book Executive Review
```

## Implementation instructions

### 1. Add a signup / access page

Create a new route:

```text
/exposure-review/start
```

Page title:

```text
Start your Free Exposure Review
```

Body copy:

```text
Create a secure Certen workspace to run a read-only Microsoft 365 exposure review. Certen will not modify licences, users, groups or tenant settings.
```

Primary form fields:

```text
Work email
Full name
Company
Role
Region
```

CTA:

```text
Create Review Workspace
```

Secondary CTA:

```text
Book Executive Review Instead
```

---

### 2. Update CTA routing

Change all public CTAs:

```text
Run Free Exposure Review
Get started
```

to route to:

```text
/exposure-review/start
```

Do **not** route unauthenticated users directly to `/connect/microsoft-365`.

---

### 3. After form submit

For now, create a lightweight local/session workspace state.

Example state shape:

```ts
type CertenWorkspaceMode = "DEMO" | "LIVE";

type CertenWorkspaceSession = {
  userEmail: string;
  fullName: string;
  company: string;
  role?: string;
  region?: string;
  mode: CertenWorkspaceMode;
  createdAt: string;
};
```

Store temporarily in:

```ts
localStorage.setItem("certen.workspace", JSON.stringify(workspace));
```

Default mode should be:

```ts
DEMO
```

Then redirect to:

```text
/exposure-review/workspace
```

# P2 — Fix Demo / Live mode switching

## Current problem

Once Demo mode is selected, the app keeps forcing Demo and the user cannot switch to Live.

## Required behaviour

The user must always be able to see and switch environment mode.

## Implementation instructions

### 1. Add global environment selector

Add this to the authenticated app header/top nav:

```text
Environment: Demo ▼
Environment: Live ▼
```

Options:

```text
Demo Workspace
Live Workspace
```

When changed, update:

```ts
localStorage.setItem("certen.workspace.mode", selectedMode);
```

---

### 2. Add logout / reset session

Add menu item:

```text
Sign out
```

On click:

```ts
localStorage.removeItem("certen.workspace");
localStorage.removeItem("certen.workspace.mode");
window.location.href = "/";
```

---

### 3. Prevent forced demo redirect

Search for logic like:

```ts
if (demoMode) redirect(...)
```

or hardcoded demo default routing.

Replace with:

```ts
const mode =
  new URLSearchParams(window.location.search).get("mode") ||
  localStorage.getItem("certen.workspace.mode") ||
  "DEMO";
```

Only use Demo data when:

```ts
mode === "DEMO"
```

Only use Live connection when:

```ts
mode === "LIVE"
```
