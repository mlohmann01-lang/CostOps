Certen Executive Command Center State Separation Fix
Objective

Remove all synthetic/demo leakage from LIVE mode.

The Executive Command Center must support four distinct runtime states:

DEMO
LIVE_UNCONNECTED
LIVE_DISCOVERING
LIVE_OPERATIONAL

The UI, metrics, findings, recommendations, value snapshots and chain stages must derive from the current runtime state.

Problem

Current behaviour:

LIVE mode displays:

No Data
Connected, but no data has been recorded yet

while simultaneously showing:

$320,000 identified
$120,000 approved
$80,000 executed
$64,000 verified

ChatGPT governance risks
Slack renewal risks
Claude findings
Tableau findings

This creates a contradiction.

If no tenant data exists, Certen must not display synthetic findings or value.

Build Requirement 1
Introduce explicit workspace runtime states

Create canonical runtime state definition.

export type WorkspaceRuntimeState =
  | "DEMO"
  | "LIVE_UNCONNECTED"
  | "LIVE_DISCOVERING"
  | "LIVE_OPERATIONAL";

Create helper:

export function getWorkspaceRuntimeState(): WorkspaceRuntimeState

Rules:

if (workspace.mode === "DEMO")
  return "DEMO";

if (
  workspace.mode === "LIVE" &&
  !connectedConnectors.length
)
  return "LIVE_UNCONNECTED";

if (
  workspace.mode === "LIVE" &&
  connectedConnectors.length > 0 &&
  discoveryComplete === false
)
  return "LIVE_DISCOVERING";

return "LIVE_OPERATIONAL";

All Executive Command Center views must use this state.

The work is complete when:

1. DEMO never appears in LIVE mode.
2. LIVE_UNCONNECTED shows zero findings.
3. LIVE_UNCONNECTED shows zero value.

5. LIVE_DISCOVERING only shows discovered value.

8. Data trust is unavailable when nothing is connected.
9. Connector attention panel shows actual connection state.
10. No synthetic evidence is rendered outside DEMO.
