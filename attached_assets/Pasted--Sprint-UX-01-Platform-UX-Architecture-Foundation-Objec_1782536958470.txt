# Sprint UX-01 — Platform UX Architecture Foundation

## Objective

The platform now contains multiple executive surfaces that have independently evolved.

This sprint establishes a single UX architecture that every remaining page will follow.

This is **not** a documentation exercise.

This is an implementation sprint.

Deliver reusable page archetypes, reusable components, and a platform page registry that future pages compose from.

Do **not** redesign every page in this sprint.

Build the foundation that allows every future redesign to be consistent.

---

# Goal

Certen should stop feeling like 20 separate pages.

It should feel like one Economic Control System viewed through different lenses.

---

# Deliverable 1 — Platform Page Registry

Create a canonical registry.

Example:

```ts
export interface PlatformPageDefinition {
  id: string;

  title: string;

  workspace:
    | "EXECUTIVE"
    | "DISCOVERY"
    | "EXECUTION"
    | "VALUE"
    | "EVIDENCE"
    | "PLATFORM";

  archetype:
    | "EXECUTIVE_DASHBOARD"
    | "LIFECYCLE_DASHBOARD"
    | "WORKFLOW_DASHBOARD"
    | "DISCOVERY_DASHBOARD"
    | "REGISTRY_DASHBOARD"
    | "OPERATIONS_DASHBOARD";

  executiveQuestion: string;

  primaryPersona:
    | "CIO"
    | "CFO"
    | "ITAM"
    | "FINOPS"
    | "OPERATOR"
    | "AUDITOR";

  primaryDecision: string;

  heroComponent: string;

  visualComponent: string;

  detailComponent: string;
}
```

Populate every existing page.

This becomes the platform source of truth.

# Deliverable 5 — Component Adoption

Do **not** rewrite pages.

Instead:

Convert the completed pages to consume the shared components.

Initially:

* Executive Command Center
* Outcome Ledger

Both should render from the shared executive component library.

Behaviour must remain identical.

Visual consistency should improve.

# Deliverable 6 — Shared Runtime Behaviour

Standardise runtime handling.

Every page should consume the same runtime primitives.

Support:

```text
DEMO

LIVE_UNCONNECTED

LIVE_DISCOVERING

LIVE_OPERATIONAL
```

Shared empty states.

Shared banners.

Shared health indicators.

No page-specific implementations.

# Deliverable 9 — Zero Functional Change

Do not change:

* business logic
* APIs
* runtime state model
* canonical schemas
* evidence model
* ledger model
* calculations

This sprint is structural only.

---

# Acceptance Criteria

Complete when:

* Platform Page Registry exists.
* Six page archetypes exist.
* Shared executive component library exists.
* Executive Command Center uses shared components.
* Outcome Ledger uses shared components.
* Sidebar reads from registry metadata.
* Runtime behaviour is shared.
* Existing behaviour is unchanged.
* Typecheck passes.
* Full test suite passes.
* Browser verified.

Run:

```bash
pnpm typecheck
pnpm test
pnpm build
```

This sprint creates the UX foundation for every remaining page redesign.

Future redesigns should primarily be composition of shared components rather than bespoke page implementations.
