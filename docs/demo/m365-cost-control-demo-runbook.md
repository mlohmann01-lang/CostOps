# M365 Cost Control Demo Runbook

## 1. Demo story
Contoso Retail uses governed execution to convert license evidence into measurable savings outcomes.

## 2. Persona
FinOps operator + IT governance approver.

## 3. Problem
License waste is visible but not consistently governed, executed, and proven.

## 4. Evidence ingestion
Use seeded M365 evidence records (safe synthetic data).

## 5. Recommendation generation
Show reclaim recommendations from disabled/inactive/underused license patterns.

## 6. Suppression evidence
Show admin/service and missing-evidence suppressions with reason codes.

## 7. Governance policy
Show policy thresholds, restricted actions/playbooks, and required approvals.

## 8. Approval queue
Show pending approval for plan eligibility. Approval changes state only.

## 9. Orchestration plan
Show plan created from READY_FOR_ORCHESTRATION recommendations.

## 10. Batch/safety controls
Show demo batch and blast-radius/risk controls.

## 11. Verification loop
Show verification records and rollback-review governance posture.

## 12. Savings proof dashboard
Show expected vs verified savings separation.

## 13. What this proves
Governed orchestration, supervised execution readiness, and proof-oriented savings outcomes.

## 14. What it deliberately does not do
No direct Graph execution, no connector runtime calls, no auto-execution in demo flow.

## Guided UI checklist
- Navigate to `/recommendations`: demonstrate evidence-backed recommendations and suppressions.
- Navigate to `/governance`: demonstrate policy controls and approval queue.
- Navigate to `/execution-orchestration`: demonstrate plan, batch controls, and supervised queue readiness.

**Core message:** This is not a reporting tool. It is a governed execution and proof layer for cost-control outcomes.
