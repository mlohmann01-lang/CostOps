# UI Coherence Review: Governed Economic Outcomes

## Goal
Ensure control-plane pages communicate one coherent story: **governed economic outcomes**.

## Page Audit

### Command / Home
- Should represent global verdict + next governed action.
- Keep language anchored on governed readiness, drift, and realized value.

### Recommendations
- Primary place for opportunity discovery and governance posture.
- Must clearly show policy blockers, trust/readiness, and projected value.

### Governance
- Should show policy state, approvals, expirations, and scheduler reevaluations.
- Keep governance controls visibly upstream of execution.

### Outcome Ledger
- Canonical economic proof destination.
- Shows verified savings, variance, pending/failed verification, drift posture, and evidence-backed outcomes.

### Connectors / Integrations
- Should communicate evidence health and execution capability boundaries.
- Explicitly separate discovery capability from execution capability.

## Coherence Recommendations (No Feature Additions)
1. Use deterministic vocabulary across pages:
   - Governed economic outcome
   - Projected savings
   - Verified savings
   - Savings variance
   - Pending verification
   - Verification failed
   - Drift detected
   - Evidence-backed
2. Ensure each page maps to a distinct lifecycle stage, avoiding repeated mixed concerns.
3. Keep call-to-action progression consistent:
   - Discover -> Govern -> Execute (gated) -> Verify -> Ledger -> Monitor drift.
